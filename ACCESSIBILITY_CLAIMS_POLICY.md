# Accessibility Claims Policy

Status: `INTERNAL_POLICY_ONLY`

This policy keeps Earnalism accessibility language truthful while the audiobook and non-visual journeys remain under evidence gathering. It does not claim WCAG compliance, blind-user testing, screen-reader certification, or full accessibility.

## Current Approved Public Position

Allowed public language today:

- Dracula is available as the approved core reading release.
- Dracula audio is not available yet.
- Reading time is used only while reading.
- Future audiobook and accessibility work is in internal QA.
- Support questions can be sent to the configured support contact.

## Blocked Public Claims

Do not publish or imply any of these claims unless the required evidence exists and owner/legal review approves the exact wording:

- `WCAG compliant`
- `blind-user tested`
- `screen-reader certified`
- `fully accessible`
- `fully accessible audiobook platform`
- `audiobooks are live`
- `Listen Now`
- `Dracula audiobook is available`
- `Kshudhita Pashan audiobook is available`
- `human narrated` unless fully human-recorded and rights-approved
- `no AI touch` if AI generation, enhancement, transcription, or review was used

## Internal Language Allowed

Internal reports may use phrases such as:

- accessibility readiness
- screen-reader readiness
- non-visual journey audit
- keyboard navigation review
- assistive-technology test script
- accessibility ambition
- evidence required before public claim

These phrases must stay in internal docs unless reviewed for public use.

## Evidence Required For Stronger Claims

| Claim type | Evidence required before public use |
| --- | --- |
| WCAG compliance | Formal scope, criteria mapping, testing evidence, issue log, remediations, owner/legal approval |
| Blind-user tested | Named test plan, participant consent/evidence handling, findings, remediations, owner/legal approval |
| Screen-reader certified | Certification authority or formal test basis, device/browser matrix, owner/legal approval |
| Fully accessible | Not allowed without legal review. Prefer specific evidence-based statements. |
| Audiobooks live | Approved audio release gate, public asset audit, rights evidence, QA, support/refund readiness, rollback approval |
| Human narration | Human recording evidence, narrator contract/consent, commercial rights, attribution, owner/legal approval |

## Safe Alternatives

Use specific, non-overclaiming language:

- `Audio is not available yet.`
- `Accessibility work is in internal QA.`
- `Manual screen-reader testing is still required before public accessibility claims.`
- `Future audiobook release requires rights, QA, accessibility, support, and owner approval.`
- `No public audiobook is live today.`

## Enforcement

The audiobook release gate and regression suite must fail if public copy claims:

- audiobooks are live before approval
- WCAG compliance without evidence
- blind-user testing without evidence
- screen-reader certification without evidence
- full accessibility without evidence
- Listen Now or public audio availability for Dracula, Kshudhita Pashan, or any unapproved title
