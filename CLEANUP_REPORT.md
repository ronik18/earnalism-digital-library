# Earnalism Production Cleanup Report

Generated: 2026-06-17

## Summary

The public cleanup focused on preventing unrelated demo/template/fashion/WooCommerce-style content from returning a healthy public `200` shell or appearing in sitemap/robots-visible public surfaces.

Repository search found no real fashion/clothing/WooCommerce CMS records in source-controlled content, but it did identify a dangerous legacy fallback:

- `/shop/:slug` redirected client-side to `/book/:slug`.
- Unknown SPA routes, including `/product/patterned-wrap-dress` and `/journal/denim-jackets`, currently return the generic React HTML shell with HTTP `200` in production until this cleanup is deployed.

## Removed Or Quarantined URL Families

The following route families are blocked from public indexing and routed to removed-content handling:

- `/product/*`
- `/products/*`
- `/product-category/*`
- `/shop/*`
- `/fashion/*`
- `/clothing/*`
- `/apparel/*`
- `/tag/fashion`
- `/tag/clothing`
- `/tag/apparel`

`/shop` and `/shop/` now redirect permanently to `/library` because the root shop URL can be interpreted as accidental legacy traffic, while product-specific shop paths are treated as removed demo/ecommerce content.

Known demo/template URLs marked for `410 Gone`:

- `/product/patterned-wrap-dress`
- `/products/patterned-wrap-dress`
- `/shop/patterned-wrap-dress`
- `/journal/denim-jackets`
- `/blog/denim-jackets`
- `/denim-jackets`
- `/fashion`
- `/clothing`
- `/apparel`
- `/product-category/clothing`
- `/tag/fashion`

No legitimate Earnalism books, reader pages, pricing pages, memberships, or journal pages were deleted.

## Redirects And 410s Added

- Added `frontend/api/removed-content.js`.
- Updated `frontend/vercel.json` so blocked demo/ecommerce/fashion routes resolve before the SPA catch-all.
- Added `/shop` and `/shop/` permanent redirects to `/library`.
- Removed the client-side `/shop/:slug -> /book/:slug` redirect.
- Removed demo pages return `410 Gone` when the path contains known blocked terms.
- Generic blocked route families return non-indexable removed content instead of the homepage shell.

## Sitemap And Robots Changes

- Updated `frontend/scripts/generate-seo-assets.mjs` with blocked prefixes and blocked terms.
- Regenerated `frontend/public/sitemap.xml`.
- Removed crawler blocks for demo/ecommerce route families from `frontend/public/robots.txt` during deindexing.
- Removed demo/ecommerce URLs remain excluded from `sitemap.xml` and rely on `410 Gone` plus `X-Robots-Tag: noindex, nofollow, noarchive`.
- Robots blocking can be reconsidered later after search indexes have dropped the retired URLs.
- Confirmed blocked terms are excluded from the generated sitemap.

## `/product/*` Policy Decision

- `/product/*`, `/products/*`, and `/product-category/*` are treated as legacy/demo WooCommerce residue for now.
- Approved Earnalism paid/book pages must use `/book/*`, `/pricing`, `/membership`, `/reading-pass`, `/institution`, or another explicitly approved non-WooCommerce route.
- If Earnalism later needs public product-style pages, add an explicit approved-product allowlist before the broad `/product/*` removal rule.

## Rollback Steps

If Vercel route handling causes unexpected blockage:

1. Revert `frontend/vercel.json`.
2. Revert `frontend/api/removed-content.js`.
3. Revert the `/shop` route removal in `frontend/src/App.js` only if legacy shop compatibility is intentionally restored.
4. Rebuild and redeploy frontend.
5. Keep `scripts/audit-public-content.mjs` and reports unless they are directly implicated.
