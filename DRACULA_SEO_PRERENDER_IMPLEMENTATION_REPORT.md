# Dracula SEO Prerender Implementation Report

Generated: 2026-06-21

## Summary

This PR keeps the existing Create React App plus Vercel architecture and hardens the current static snapshot approach for crawler-visible Dracula SEO. The implementation keeps Dracula as the only approved public reading release, keeps `/reader/dracula` noindex/canonicalized to `/book/dracula`, and preserves tombstoned legacy ecommerce/demo routes.

No deployment, production mutation, payment action, audiobook enablement, provider call, or public publishing action was performed.

## Architecture Chosen

Chosen approach: postbuild static SEO snapshots generated by `frontend/scripts/generate-static-seo-snapshots.mjs`.

Reason:
- It is already compatible with the CRA/Vercel setup.
- It produces route-specific raw HTML before client hydration.
- It avoids a framework migration or SSR runtime change.
- It can be validated locally by raw HTML audits and regression tests.

Vercel serves generated files such as `build/book/dracula/index.html`, `build/library/index.html`, `build/pricing/index.html`, and `build/reader/dracula/index.html` before the SPA fallback, while retired legacy routes continue to route to the removed-content handler.
